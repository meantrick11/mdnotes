# 双指针对应技巧

## 一、滑动窗口（Sliding Window）

### 1. 基本定义

滑动窗口是一种**双指针技巧**，用于解决**连续子数组或子串**的优化问题。它维护一个左右边界 `[left, right]` 的动态窗口，通过**同向移动**两个指针来遍历所有可能的连续区间，将暴力枚举的 O(n²) 优化到 O(n)。

- **双指针关系**：滑动窗口属于双指针类的**同向指针**子类（快慢指针的一种特殊形式）。
- **常见别名**：虫取法、追逐指针。

### 2. 适用场景（关键词）

- 子数组 / 子串 **连续**
- 求 **最大/最小长度**
- 满足某个条件（和 ≥ target、包含所有字符、最多 k 个不同字符等）
- 数组元素通常为正数（若含负数需结合前缀和+哈希）
- 固定长度窗口 or 可变长度窗口

### 3. 核心思想

1. 右指针 `right` 不断右移，**扩大窗口**，加入新元素。
2. 当窗口满足条件后，尝试**收缩左边界** `left`，寻找更优解。
3. 每次收缩前更新答案（如最小长度）。
4. 收缩到条件不再满足时，继续移动右指针。

> 关键：每个元素最多进、出窗口各一次，总复杂度 O(n)。

### 4. 通用模板（可变长度，求最小窗口）

```python
def min_window(nums, condition):
    left = 0
    cur_state = 0          # 窗口和、计数、频率表等
    ans = len(nums) + 1    # 初始化为不可能的大值

    for right in range(len(nums)):
        # 1. 扩展窗口：加入 nums[right]
        cur_state += nums[right]   # 或其他更新逻辑

        # 2. 收缩窗口：当条件满足时，尝试缩小
        while cur_state >= target:   # 条件视题目而定
            ans = min(ans, right - left + 1)   # 更新答案
            cur_state -= nums[left]            # 移除左边界元素
            left += 1

    return ans if ans <= len(nums) else 0
```

**不同题型模板变形**：
- **求最大窗口**：通常在 while 不满足条件时收缩，在循环外更新答案。
- **固定长度窗口**：先建窗口，然后整体滑动（无需 while 收缩）。

### 5. 典型例题：LeetCode 209. 长度最小的子数组

**题目**：给定正整数数组 `nums` 和目标值 `target`，找出和 ≥ target 的最短连续子数组长度。  
**输入**：`target = 11, nums = [1,2,3,4,5]`  
**输出**：`3`（子数组 `[3,4,5]` 或 `[2,3,4,5]`）

#### 手写模拟

| right | 操作 | 窗口 | 当前和 | 是否 ≥11 | 收缩动作 | 更新长度 |
|-------|------|------|--------|----------|----------|----------|
| 0 | 加1 | [0,0] | 1 | 否 | 无 | - |
| 1 | 加2 | [0,1] | 3 | 否 | 无 | - |
| 2 | 加3 | [0,2] | 6 | 否 | 无 | - |
| 3 | 加4 | [0,3] | 10 | 否 | 无 | - |
| 4 | 加5 | [0,4] | 15 | 是 | 移出1 → 和14，窗口[1,4]；移出2 → 和12，窗口[2,4]；移出3 → 和9，停止 | 长度5→4→3，最小=3 |

最终返回 `3`。

#### 正确代码

```python
class Solution:
    def minSubArrayLen(self, target: int, nums: List[int]) -> int:
        n = len(nums)
        ans = n + 1
        left = 0
        cur_sum = 0
        for right in range(n):
            cur_sum += nums[right]
            while cur_sum >= target:
                ans = min(ans, right - left + 1)
                cur_sum -= nums[left]
                left += 1
        return ans if ans <= n else 0
```

### 6. 常见错误（结合你的原始代码总结）

| 错误操作 | 后果 | 正确做法 |
|---------|------|----------|
| 收缩条件使用 `(sum - nums[left]) > target` | 窗口未充分收缩，可能遗漏更优解 | 使用 `while sum >= target` |
| 更新答案时用 `right - left` | 少计1，答案偏小 | 用 `right - left + 1` |
| 错误覆盖累加和 `sum = nums[left]` | 窗口状态错乱 | 使用 `sum -= nums[left]` |
| 将答案更新放在收缩循环外面 | 只能记录某个特定时刻的长度 | 在收缩循环内部（每次移除前）更新 |
| 忘记处理无解情况 | 返回不合理值 | 最后判断 `ans` 是否被更新 |

### 7. 滑动窗口 vs 其他双指针

| 类型 | 指针移动方向 | 典型例题 |
|------|-------------|----------|
| **滑动窗口** | 同向（右快左慢） | 最小覆盖子串、最长无重复子串 |
| **快慢指针** | 同向（不同速度） | 环形链表、删除有序数组重复项 |
| **对撞指针** | 相向（两端向中间） | 两数之和 II、回文判断 |

### 8. 复杂度分析

- **时间复杂度**：O(n) —— 每个元素最多被 `left` 和 `right` 各访问一次。
- **空间复杂度**：O(1) 或 O(k)（k 为字符集大小，如哈希表存储频率）。

### 9. 实战提醒

- 窗口条件必须**单调**（如和、长度等），否则双指针失效（负值场景需特殊处理）。
- 当数组元素**全为正数**时，滑动窗口求“和 ≥ target 的最短子数组”正确性有保障。
- 对于“最长无重复子串”问题，收缩条件变为 `while 窗口内有重复字符`，需配合哈希表。

**最后一句记忆口诀**：
> 右指针扩张进元素，左指针收缩找最短；  
> 条件一满足即更新，每个元素只走一遍。
