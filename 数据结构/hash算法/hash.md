# hash 简介

## 基本概念

**hashmap**是一种通过键(key)直接访问值(value)的数据结构。它利用一个叫做hash函数的公式，将一个任意的键(str,int...)转化为一个固定大小的数字，这个数字被称为hash值或者hash码，然后利用这个hsh值作为数组的索引，将值存储在对应的数组位置上。

*python中的字典就是一种hash表的具体实践*

## 核心组件
一个hashmap主要有以下三个核心部分组成：
1. 键(key):你要存储和查找数据的标识符
2. 值(value):与键相关联的实际数据
3. hash函数:将键映射到数组索引的数学函数

## 常见hash构造方法
| 类型         | 示例                                       | 适用场景       |
| ------------ | ------------------------------------------ | -------------- |
| 除法取余法   | hash = key % table_size                    | 整数键         |
| 乘法取整法   | hash = floor(key * A % 1 * table_size)     | 浮点数键       |
| 数字分析法   | 分析数字的分布规律                         | 特定模式数据   |
| 平方取中法   | hash = (key * key) // 10^(n/2) % table_size| 数字键         |
| 字符串哈希   | 逐字符处理                                 | 字符串键       |

## hash函数的特性以及举例

### hash函数的特性

1. 确定性:相同的键必须始终产生相同的hash值
2. 高效性：计算速度要快
3. 均匀分布：尽可能地将不同的键均匀地映射到整个数组空间，以减少冲突

## hash 冲突

从刚刚的简单例子中，可以知道:如果两个字符串中的ASCII码之和相等，那么它们将会被映射到同一个hash值，这就是**hash冲突**,即两个或者多个键经过hash函数计算之后，得到了相同的数组索引。解决hash冲突有经典的两种方式：


### 方法一：链地址法

这是最常用的方法，它不把数据直接放在莫格数组索引上，而是在索引位置放置一个链表(或一种其他的数据结构)，当冲突发生时，新的键值对就被添加到这个位置的链表中

过程描述:
如果该索引位置的链表均为空，那么创建一个新的节点存储键值对。
如果链表不为空（发生冲突），则遍历这个链表：
- 如果找到相同的键，则更新其值
- 如果没找到相同的键，则将新的键值对添加到链表末尾

### 方法二：开放地址法

这种方法将所有的元素直接存储在数组本身，当冲突发生时，它会按照魔种探测序列在数组中寻找下一个空闲的位置，直至找到，如果数组满了，那么就必须得做扩容处理，无法像链地址法那样'几乎无限插入'。
常见的探测方法：
- 线性探测：如果位置i被占，则尝试探测i+k,i+2k...（k表示步长）直至找到空位。
- 二次探测：如果位置i被占，则尝试i+1^2,i+2^2,i+3^2...。
- 双重hash：使用第二个hash来计算探测步长，寻找未被占用的空位
